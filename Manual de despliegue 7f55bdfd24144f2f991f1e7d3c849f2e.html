Nombre del software

ArquiSW

Alcance

El objetivo de este manual es brindarle al usuario una descripción detallada de todo el proceso que debe seguir para ejecutar los proyectos en su máquina local. Se explicarán los recursos necesarios, se le guiará a través del proceso de instalación, ejecución y adaptación de los proyectos.

Recursos de hardware

Se va a trabajar sobre un sistema operativo Windows. Se requiere un computador mínimo con 8 GB de memoria RAM, la capacidad de almacenamiento no requiere mucha atención, por lo que con la mínima capacidad la aplicación funciona (128gb). Y se requiere un computador con un buen procesador que sea capaz de ejecutar un proyecto backend y frontend al mismo tiempo.

Recursos de software

Para la ejecución del backend necesitamos instalar las siguientes herramientas:

  • PostgreSQL
  • pgAdmin 4
  • IntelliJ IDEA
  • Visual Studio Code
  • Insomnia / Postman para hacer peticiones de tipo Rest
  • Git
  • Versión 11 de Java

Para la ejecución del Frontend necesitamos las siguientes herramientas:

  • Visual Studio Code
  • Angular CLI versión 15 o superior
  • NodeJS versión 18 o superior

Configuración del producto

Aquí se debe de tener mucho cuidado cuando se está haciendo la instalación del PostgreSQL y la instalación con pgAdmin 4, se debe de tener presente la contraseña que se le va a pedir a poner a la base de datos que luego se va a estar requiriendo para cualquier modificación de la misma en cuanto a las tablas, querys, script y demás validaciones que se requieren hacer.

Para adecuar el software y ponerlo a funcionar se deben realizar los siguientes procedimientos:

  1. Se deben clonar el código desde los repositorios para descargarnos el backend y el frontend de la aplicación, de los siguientes links:

    /* Repositorio del frontend */

    CREATE TABLE nota_caracteristica_requisito 
( 
    id                  SERIAL PRIMARY KEY, 
    requisito_id        integer NOT NULL, 
    caracteristica_id   integer NOT NULL, 
    nota_caracteristica double PRECISION, 
    FOREIGN KEY (requisito_id) REFERENCES requisito (id), 
    FOREIGN KEY (caracteristica_id) REFERENCES caracteristica (id), 
    UNIQUE (requisito_id, caracteristica_id) 
); 


CREATE TABLE tipo_error_caracteristica 
( 
    id                SERIAL PRIMARY KEY, 
    requisito_id      integer NOT NULL, 
    caracteristica_id integer NOT NULL, 
    tipo_error_id     integer NOT NULL, 
    dde               boolean, 
    dii               boolean, 
    var               boolean, 
    FOREIGN KEY (requisito_id) REFERENCES requisito (id), 
    FOREIGN KEY (caracteristica_id) REFERENCES caracteristica (id), 
    FOREIGN KEY (tipo_error_id) REFERENCES tipo_error (id), 
    UNIQUE (requisito_id, caracteristica_id, tipo_error_id) 
);

  1. Creación de la función insertar_datos_iniciales
    CREATE OR REPLACE FUNCTION insertar_datos_iniciales()
    RETURNS TRIGGER AS $$
BEGIN
INSERT INTO public.nota_caracteristica_requisito (requisito_id, caracteristica_id, nota_caracteristica)
VALUES (NEW.id, 1, 0),
       (NEW.id, 2, 0),
       (NEW.id, 3, 0),
       (NEW.id, 4, 0),
       (NEW.id, 5, 0),
       (NEW.id, 6, 0),
       (NEW.id, 7, 0),

       (NEW.id, 9, 0);

INSERT INTO public.tipo_error_caracteristica (requisito_id, caracteristica_id, tipo_error_id, dde, dii, var)
VALUES (NEW.id, 1, 1, false, false, false),
       (NEW.id, 2, 1, false, false, false),
       (NEW.id, 3, 1, false, false, false),
       (NEW.id, 4, 1, false, false, false),
       (NEW.id, 5, 2, false, false, false),
       (NEW.id, 6, 2, false, false, false),
       (NEW.id, 7, 2, false, false, false),
       (NEW.id, 8, 1, false, false, false),
       (NEW.id, 9, 1, false, false, false);

INSERT INTO public.operacion (caracteristicas_evaluadas, nivel_adecuacion, nivel_caracteristica_puntuacion, promedio_calculado, puntaje_maximo, requisito_id)
VALUES (0, 0, 0, 0, 0, NEW.id);

RETURN NEW;
END
$$ LANGUAGE plpgsql;

  1. Creación del trigger para llamar la función creada insertar_datos_iniciales
